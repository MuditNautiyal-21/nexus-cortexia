# Thorough profile

Maximum quality. Higher token spend, but catches more issues and produces
fewer bugs. Use for production systems, security-sensitive code, or anything
where a bug would be expensive.

## Overrides

- Full consensus (with debate if needed) for every non-trivial task
- Two-stage review for all tasks including trivial ones
- Tester agent added to team regardless of project size
- Write tests before implementation (TDD: red-green-refactor)
- Dependency outputs compressed to ~500-1000 tokens (more detail preserved)
- Security-focused review pass in addition to standard review
- Post-integration end-to-end verification step
- Target: 150-200% of standard profile token cost
